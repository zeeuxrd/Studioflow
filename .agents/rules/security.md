# Security & Compliance – StudioFlow

## Core Principles (from PRD)

- **User owns all generated content** – AI output is non-exclusive and reusable by the system.
- **No personal data shared externally** – niche and behavioral data stored securely.
- **No tracking outside product scope** – no analytics pixels, no third-party cookies.
- **GDPR-aligned handling** – user export rights, right to deletion.

## Content Ownership

- Every exported piece of content must include a visible disclaimer:
  > "Generated by StudioFlow AI. You own this content. The AI's output is non-exclusive."
- Users can download all their data (ideas, posts, products) as a JSON file.
- System cannot resell or republish user-generated content without explicit permission.

## Data Privacy

### Data Collected
- User profile (niche, tone, platform focus, monetization goal) – optional, can be skipped.
- Generated ideas, posts, products – stored per session.
- Behavioral signals (e.g., which content user chooses to productize) – only for nudges, not sold.

### Data Storage
- In Google Antigravity session memory only – no persistent database unless user opts in.
- If persistent storage added, must encrypt at rest using AES-256.
- User IDs are pseudonymous (no PII like email required).

### Data Deletion
- Users can request deletion via `DELETE /user-data` – all associated objects removed within 30 days.
- Retention Engine does not store historical data beyond 90 days.

## Monetized Content Safety

- **No false income claims** – AI-generated price suggestions must include disclaimer:
  > "This is an estimate. Actual earnings may vary."
- Before publishing a monetized product, system warns:
  > "You are about to publish an AI-generated product. Ensure it meets your quality standards and does not impersonate others."
- Prohibited product types: get-rich-quick schemes, fake engagement generators, impersonation kits.

## Spam & Abuse Prevention

- Limit batch generation: max 20 ideas per hour, max 10 product exports per day.
- Require explicit confirmation before bulk exports (>5 items).
- Rate-limit regeneration requests: allow 3 per minute, then show "Refine instead of regenerate".

## Edge Case Security

### Vague or Harmful Inputs
- If user inputs a niche involving hate speech, self-harm, or illegal activities → block generation and return:
  > "Cannot generate content for this topic. Please choose another niche."
- Log attempt anonymously for abuse monitoring.

### AI Hallucination / False Attribution
- If AI generates a quote or statistic, must prepend `[AI suggestion]` or include placeholder `[verify source]`.
- No impersonation of real people (celebrities, influencers) without clear parody label.

## Compliance Checklist

- [x] GDPR right to access – user can export all data.
- [x] GDPR right to erasure – deletion endpoint.
- [x] No hidden monetization logic – all price suggestions are explicit.
- [x] Transparent AI labeling – every AI output has `generated_by: "StudioFlow AI"` metadata.
- [x] No fake engagement generation – system never generates fake likes, comments, or followers.

## Security Recommendations for Google Antigravity

- Use environment variables for any API keys (none currently, but future integrations with Stripe/Gumroad must be secret).
- Validate all user inputs with regex to prevent prompt injection (e.g., block `"ignore previous instructions"` patterns).
- Set session timeout: 30 minutes of inactivity → auto-save draft ideas.
- Implement CORS policies if exposed via web – allow only your domain.
